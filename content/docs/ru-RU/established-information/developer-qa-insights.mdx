# Вопросы и ответы разработчиков, технические идеи

Эта обширная сессия вопросов и ответов с разработчиками Hytale раскрывает важные технические детали об архитектуре игры, философии моддинга, игровых системах и подходе к разработке.

## Техническая архитектура

### Сеть и инфраструктура

* **Протокол**: использует QUIC (надежный UDP с ненадежными каналами) вместо TCP
* **Совместимость с сервером**: Выполняется на любой платформе, поддерживающей Java 25
* **Метод соединения**: Прямой IP, обнаружение сервера после запуска
* **Слой данных**: Текущий собственный формат сериализации будет заменён после релиза.
* **Производительность**: Многоядерная поддержка - основной поток + параллельное выполнение в каждом мире
* **Рендеринг**: OpenGL 3.3 для совместимости Mac, потенциальная будущая миграция на Vulkab/Metal

### Безопасность и распространение

* **Без выполнения кода на стороне клиента**: Предотвращает использование уязвимостей для удаленного выполнения кода.
* **Доступные для загрузки данные**: Только конфигурационные файлы и ассеты.
* **Распределение модов**: платформы сообщества (CurseForge, Modrinth), нет непосредственного внутриигрового маркетплейса
* **Хостинг сервера**: доступно с первого дня, никаких деловых соглашений не требуется (только лицензионное соглашение с конечным пользователем/условия предоставления услуг)

## Возможности моддинга

### Мощьность со стороны сервера

* **Полный доступ к Java**: соединения с базами данных, веб-запросы, любая функциональность Java
* **Пользовательские библиотеки**: Позволяет создавать скрипты Lua, ML фреймворки или любую другую Java библиотеку
* **Модификация сервера**: Общий исходный код позволяет жестко модифицировать сервер
* **Горячая перезагрузка**: поддерживается для большинства типов ассетов
* **Документация**: планируется публичный GitBook, хотя изначально он будет немногословен.

### Создание контента (кодинг не требуется)

* **Создание мира**: Пользовательские биомы и карты без Java
* **NPC и предметы**: Конфигурация на основе JSON
* **Рецепты крафта**: Система, управляемая данными
* **Визуальный скриптинг**: Заменяет командные блоки, полностью изолировано
* **Система сборных конструкций**: сложные строительные блоки с внутриигровым редактором

### Продвинутый моддинг

* **Боевая система**: Система взаимодействия на основе данных для цепочек атак и способностей
* **Дополнительные хитбоксы**: Поддержка сложной механики боссов (требуется улучшение)
* **Статусные эффекты**: Система усилений/ослаблений с пользовательскими эффектами и характеристиками.
* **Кастомный пользовательский интерфейс**: переход на NoesisGUI обеспечивает мощную настройку интерфейса
* **Физика блоков**: Избирательная система (деревья падают, а поставленные блоки — нет)

## Геймплейные системы

### Основные Механики

* **Система выносливости**: Без голода по умолчанию (Может быть изменено)
* **Гравитация**: Физика отдельных блоков — деревья рушатся, здания могут парить.
* **Освещение**: Крупная переработка для исправления несоответствий между моделями и вокселями.
* **Высота мира**: Сейчас ограничена, может измениться с обновлением системы хранения вокселей.
* **Автоматизация**: Га релизе не будет аналога редстоуна (можно будет добавить модами).

### Система Игрока

* **Модели персонажей**: Человек по умолчанию, можно изменить в творческом режиме
* **Аватары в мультиплеере**: Управляется сервером (нет пользовательских моделей в мультиплеере)
* **Скины**: Возможна полная кастомизация ("Можешь превратить себя в картошку :D")
* **Шейпешифтинг**: Зелье трансмутации для веселья/моддинга

### Мир и NPC

* **Фракционная система**: динамическое взаимодействие с NPC
* **Система жителей**: Планируется динамические деревни с отношениями и поведением жителей
* **Загрузка чанков**: Сущности выгружаются, когда выходят за пределы дальности рендеринга всех игроков.
* **Сиды**: Поддерживаемые сиды для генерации мира

## Философия разработки

### Стратегия релиза

* **Приоритет базовой игры**: Режим приключений — основной фокус, архитектура моддинга его поддерживает.
* **Технический долг**: Признан необходимым для релиза, есть обязательство по его дальнейшему сокращению и улучшению системы.
* **Частота обновления**: Планируется очень часто обновляться, особенно после запуска
* **Политика версий**: Всегда последняя версия, но серверы могут запускать модифицированные версии

### Фокус сообщества

* **Поддержка Создателей**: Запланированные контракты для участников сообщества
* **Обучение**: Запланированные с партнерами официальные уроки по моддингу
* **Интеграция отзывов**: Твердая приверженность улучшениям, инициированным сообществом
* **Доступность**: Разработано для аппаратного обеспечения начального уровня, разумные системные требования

### Инструменты и документация

* **Интеграция с Blockbench**: заменяет пользовательский "Hytale Model Maker".
* **Инструменты Machinima**: Полные возможности создания кинотеатров
* **Инструменты для творчества**: запланировано крупное обновление функций строительства/творчества
* **Доступ к исходному коду**: Код сервера будет раскрыт с помощью комментариев разработчиков.

## Поддержка платформ

* **Текущий**: Windows (основной)
* **Планируется**: Поддержка Mac/Linux активно в разработке
* **VR**: Нереализуемо в обозримом будущем
* **Производительность**: неограниченная частота кадров, разработано с учётом доступности.
* **Оффлайн игра**: Однопользовательский режим работает без подключения к Интернету

## Монетизация и юридические вопросы

* **БЕЗ Pay-to-Win**: твёрдое обязательство не использовать P2W-механики.
* **Только косметики**: недорогие косметические наборы для поддержки инфраструктуры
* \**EULA серверов*: условия для политики монетизации серверов находятся на этапе финальных правок.
* **Блокировка QUIC**: Подтверждённая проблема в некоторых регионах, без альтернативных протоколов

## Текущие ограничения

* Пользовательские шейдеры: не поддерживаются до тех пор, пока не будет найдено решение для песочницы.
* **Ограничения UI/ввода**: расширяются, но в данный момент остаются ограниченными.
* Документация: поначалу будет скудной, со временем улучшится.
* **Клиентские моды**: код не выполняется на стороне клиента для безопасности
* **Таблички с именами**: на данный момент возможности кастомизации ограничены.

Эта всесторонняя техническая основа делает Hytale безопасной и расширяемой платформой, которая предоставляет создателям широкие возможности, сохраняя при этом производительность, доступность и стремление к постоянному развитию.
