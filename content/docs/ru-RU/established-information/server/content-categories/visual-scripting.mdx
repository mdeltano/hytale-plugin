# Визуальный скриптинг

Hytale планирует поддерживать визуальное написание скриптов как метод, позволяющий сделать расширенную логику и управление потоками доступными для более широкой аудитории. Эта система не будет доступна при запуске, но будет доступна вскоре после запуска.

## Обзор

Визуальный скриптинг в Hytale предназначен для работы аналогично чертежам в Unreal Engine. Он позволяет дизайнерам создавать сложное поведение через визуальный интерфейс, который соответствует тому, как они представляют игровую логику, без необходимости писать код напрямую.

### Ключевая особенность

* **Подмножество функциональности Java**: Визуальный скриптинг может выполнять лишь часть того, что возможно в Java-плагинах.
* **Удобно для дизайнеров**: «Дизайнеры получают возможность работать с визуальным языком, который соответствует их представлениям о поведении».
* **Расширяемо программистами**: разработчики могут расширять и оптимизировать систему, добавлять новые визуальные ноды и реализации для чувствительных к производительности задач
* **Безопасное совместное использование среды**: Обеспечивает безопасную среду для обмена логикой между мододелами.

## Преимущества над традиционным кодом

Визуальное написание скриптов делают пользовательскую логику более доступной, удаляя обычные препятствия найденные при разработке на Java:

* **Не существует сложных настроек окружения**: Нет необходимости беспокоиться об установке правильных инструментов или настройке среды разработки
* **Syntax-free development**: устраняет необходимость изучать и запоминать синтаксис программирования.
* **Безопасное совместное пользование**: обнаружение вредоносного ПО в визуальных скриптах практически невозможно
* **Более инклюзивно**: снижает порог входа для людей без навыков программирования

## Связь с Java

Визуальный скриптинг и Java плагины разработаны для совместной работы:

* **Java расширяет визуальный скриптинг**: программисты могут создавать новые визуальные ноды при помощи Java
* **Оптимизация производительности**: Хотя визуальный скриптинг может быть не таким оптимизированным, как Java напрямую, программисты могут реализовать критичную по производительности логику на Java и представить её в виде визуальных нодов.
* **Лучшее из обоих миров**: Позволяет оставить большую часть логики в доступной среде визуального скриптинга, сохранив при этом оптимизацию ресурсозатратных операций.

Hytale настоятельно рекомендует визуальный скриптинг, поскольку он является более доступным и безопасным для обмена и скачивания. Однако полностью работать на Java всё равно будет возможно для тех, кому это предпочтительнее.

## Визуальный 3D скриптинг

Одной из уникальных особенностей Hytale является внедрение визуального скриптинга прямо в 3D мир:

* **Редактирование в мире**: связывайте рычаги, двери, спавнеры, триггеры и другие элементы прямо внутри игры
* **Создание приключенческих карт**: отлично подходит для создания приключенческих карт и событий, не выходя из Hytale
* **Похоже на DOOM SnapMap**: обеспечивает знакомый рабочий процесс для тех, кто использовал подобные системы
* **Полностью интегрировано**: работает бесшовно с ассетами и стеком скриптинга

## Никаких текстовых скриптов

Hytale приняла осознанное решение не включать текстовые языки скриптинга. Причины:

* **Ложный компромис**: текстовый скриптинг часто вводят, чтобы "помочь не-программистам", но он всё равно требует изучения концепций программирования
* **Рост сложности**: программисты в итоге жонглируют двумя языками, при этом один из них гораздо слабее
* **Всё ещё требует синтаксиса**: дизайнерам всё равно приходится изучать синтаксис и концепции программирования
* **Менее инклюзивно**: не решает проблему доступности по-настоящему
* **Фрагментация**: избегает "раздробления логики между несколькими 'полу-языками'"

Вместо этого Hytale считает, что визуальный скриптинг — гораздо лучший подход, который действительно делает игровую логику доступнее для широкой аудитории, сохраняя при этом мощность и гибкость, необходимые для разработки
