A Programação Orientada a Objetos (POO) é a base do Java e dos mods de Hytale Ao invés de deixar variáveis e métodos soltos pelo código, nós os organizamos em **classes** e **objetos**.

## O que é uma Classe?

Uma classe é um molde para criar objetos. Pense nisso como uma receita ou um modelo.

```java
public class Jogador {
    // Propriedades (dados)
    String nome;
    int vida;
    int nivel;
    
    // Comportamento (métodos)
    public void receberDano(int dano) {
        vida -= dano;
        System.out.println(nome + " recebeu " + dano + " de dano!");
    }
}
```

## O que é um Objeto?

Um objeto é uma instância criada de alguma classe. Se uma classe é um molde, um objeto é o resultado criado a partir desse molde ou modelo.

```java
public class Main{
    public static void main(String[] args) {
        // Cria objetos da classe Jogador
        Jogador jogador1 = new Jogador();
        jogador1.nome = "Alice";
        jogador1.vida = 100;
        jogador1.nivel = 5;
        
        Player jogador2 = new Jogador();
        jogador2.nome= "Bob";
        jogador2.vida = 80;
        jogador2.nivel = 3;
        
        // Use the objects
        jogador1.receberDano(20);  // Alice recebeu 20 de dano!
        jogador2.receberDano(15);  // Bob recebeu 15 de dano!
    }
}
```

<Callout type="info" title="Classe vs. Objeto">
  **Classe** = Diagrama (a ideia de um jogador)
  **Objeto** = Coisa real (Alice, Bob, jogadores específicos)

  Uma classe pode criar vários objetos, assim como uma receita pode fazer vários bolos!
</Callout>

## Criando uma Classe Simples

Vamos criar uma classe `Espada` para o Hytale:

```java
public class Espada {
    // Propriedades
    String nome;
    int dano;
    int durabilidade;
    
    // Método para usar a espada
    public void ataque(String alvo) {
        System.out.println(alvo + " recebeu " + damage + " de dano!");
        durabilidade -= 1;
        
        if (durabilidade <= 0) {
            System.out.println(nome + " quebrou!");
        }
    }
    
    // Método para mostrar a informação
    public void obterInfo() {
        System.out.println("Arma: " + nome);
        System.out.println("Dano: " + dano);
        System.out.println("Durabilidade: " + durabilidade);
    }
}
```

Usando a classe:

```java
public class Main {
    public static void main(String[] args) {
        Espada espada = new Espada();
        espada.nome = "Espada de Ferro";
        espada.dano = 15;
        espada.durabilidade = 3;
        
        espada.mostrarInfo();
        espada.ataque("Zumbi");
        espada.ataque("Esqueleto");
        espada.ataque("Aranha");  // Isso reduzirá a durabilidade da espada, ou a quebrará.
    }
}
```

## Construtores

Ao invés de configurar as propriedades uma por uma, use um **construtor** inicializar objetos:

```java
public class Espada {
    String nome;
    int dano;
    int durabilidade;
    
    // Construtor
    public Espada(String armaNome, int armaDano, int armaDurabilidade) {
        nome = armaNome;
        dano = armaDano;
        durabilidade = armaDurabilidade;
    }
    
    public void ataque(String alvo) {
        System.out.println(alvo + " recebeu " + dano + " de dano!");
        durabilidade--;
    }
}
```

Criar espadas se tornou mais fácil:

```java
public class Main {
    public static void main(String[] args) {
        // Com mais clareza!
        Espada espadaFerro = new Espada("Espada de Ferro", 15, 100);
        Espada espadaDiamante = new Espada("Espada de Diamante", 25, 200);
        
        espadaFerro.ataque("Zumbi");
        espadaDiamente.ataque("Chefe");
    }
}
```

<Callout type="info" title="Regras do Construtor">
  * O mesmo nome que a classe
  * Nenhum tipo de retorno (nem mesmo `void`)
  * Chamado automático ao usar `new`
  * Pode ter vários construtores (sobrecarga)

  ```java
  public class Item {
      String nome;
      int valor;
      
      // Construtor com todos os parâmetros
      public Item(String nome, int valor) {
          this.nome = nome;
          this.valor = valor;
      }
      
      // Construtor com apenas o nome
      public Item(String nome) {
          this.nome = nome;
          this.valor = 0;  // Valor padrão
      }
  }
  ```
</Callout>

## A palavra-chave `this`

A `this` refere-se ao objeto atual. Use-a para esclarecer quando os nomes dos parâmetros coincidirem com os nomes das propriedades:

```java
public class Jogador {
    String nome;
    int vida;
    
    public Jogador(String nome, int vida) {
        this.nome = nome;      // this.nome = a propriedade
        this.vida = vida;  // nome = o parâmetro
    }
}
```

Sem a `this`, o Java fica confuso:

```java
public Jogador(String nome, int vida) {
    nome = nome;      // ❌ Qual nome? Não especificado!
    vida = vida;  // ❌ Qual vida? Não especificada!
}
```

### Classe "monstro"

```java
public class Monstro {
    String nome;
    int vida;
    int ataque;
    boolean ehHostil;
    
    public Monstro(String nome, int vida, int ataque, boolean ehHostil) {
        this.nome = nome;
        this.vida = vida;
        this.ataque = ataque;
        this.ehHostil = ehHostil;
    }
    
    public void receberDano(int dano) {
        vida -= dano;
        System.out.println(nome + " recebeu " + dano + " de dano!");
        
        if (vida <= 0) {
            System.out.println(nome + " foi derrotado!");
        } else {
            System.out.println(nome + " tem " + vida + " de vida restante.");
        }
    }
    
    public int atacarJogador() {
        if (ehHostil && vida > 0) {
            System.out.println(nome + " ataca causando " + ataque + " de dano!");
            return ataque;
        }
        return 0;
    }
    
    public boolean estaVivo() {
        return vida > 0;
    }
}
```

```java
public class Main {
    public static void main(String[] args) {
        Monstro zumbi = new Monstro("Zumbi", 50, 10, true);
        Monstro galinha = new Monstro("Galinha", 10, 0, false);
        
        zumbi.receberDano(20);        // Zumbi recebeu 20 de dano!
        int dano = zumbi.atacarJogador();  // Zumbi ataca causando 10 de dano!
        
        if (zumbi.estaVivo()) {
            System.out.println("O monstro ainda está vivo!");
        }
    }
}
```

### Classe "bloco"

```java
public class Bloco {
    String tipo;
    int x, y, z;
    boolean ehSolido;
    int dureza;
    
    public Bloco(String tipo, int x, int y, int z, boolean solido, int dureza) {
        this.tipo = tipo;
        this.x = x;
        this.y = y;
        this.z = z;
        this.solido = solido;
        this.dureza = dureza;
    }
    
    public void quebrarBloco() {
        System.out.println("Quebrando bloco de " + tipo + " em (" + x + ", " + y + ", " + z + ")");
        System.out.println("Dureza: " + dureza);
    }
    
    public String mostrarPosicao() {
        return "(" + x + ", " + y + ", " + z + ")";
    }
    
    public boolean podeAtravessar() {
        return !ehSolido;
    }
}
```

## Modificadores de Acesso (Prévia)

Você viu `public`? Significa "qualquer um pode acessar isso". Aprenderemos mais sobre controle de acesso depois, mas aqui está uma prévia:

```java
public class Example {
    public String publicVar;      // Anyone can access
    private String privateVar;    // Only this class can access
    // (no modifier) String defaultVar; // Package access
}
```

Por enquanto use `public` para tudo. Nós vamos aprender quando usar o `private` no próximo artigo

<Callout type="info" title="Por que usar Classes?">
  Classes ajudam a você:

  * **Organizar** os dados e métodos relacionados em conjunto.
  * **Reutilizar** o código facilmente (crie vários objetos a partir de uma classe).
  * **Modelar** coisas do mundo real (jogadores, itens, monstros)
  * **Manter** o código (alterações em um local afetam todos os objetos).

  Sem classes, gerenciar 100 jogadores exigiria 100 variáveis separadas para cada propriedade. Com classes, é apenas 100 objetos Jogador!
</Callout>

## Exercícios Práticos

1. **Crie uma classe `Poção`**
   * Propriedades: nome, valorVida, usos
   * Construtor para definir todas as propriedades
   * Método `drink()` que cura e diminui os usos
   * Método `estaVazio()` que retorna verdadeiro se usar menos ou igual a 0

2. **Crie uma classe `Baú`**:
   * Propriedades: estaBloqueado, quantidadeItem, capacidade
   * Construtor
   * Método `adicionarItem()` que verifica a capacidade
   * Método `desbloquear()` que define a propriedade `estaBloqueado()` para falsa
   * Método `estaCheio()` que verifica se a propriedade `quantidadeItem` >= `capacidade`

3. **Crie uma classe `Aldeão`**
   * Propriedades: nome, profissão, quantiaDeTroca
   * Construtor
   * Método `saudacao()` que printa uma saudação
   * Método de `troca()` que aumenta quantiaDeTroca
   * Método `mostrarInfo()` que exibe todas as propriedades

4. **Crie múltiplos objetos**: Usando qualquer classe que você fez, crie 3 objetos diferentes e teste todos os seus métodos.
